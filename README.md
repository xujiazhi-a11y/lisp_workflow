# Lisp Workflow

用 Lisp 重新理解 AI Workflow 的一次尝试。

## 缘起

大模型落地时往往离不开多步骤的工作流编排，而当下的主流做法是拖拉拽的可视化画布。画布上的连线固然直观，但当节点数量攀升，维护成本也随之急剧上升。

这个项目尝试换一个思路：用代码（而不是连线）来表达工作流。具体来说，是借助 Lisp 及其函数式编程的思想，将二维的连线图折叠成一维的极简代码。

## 核心理念

**过程即变量**。在 Lisp 中，任何复杂的过程都可以被抽象成一个简单的函数名。Workflow 的节点本质上就是"过程"，而 Lisp 把"过程"当成数据自由传递的特性，使其天然具备承载工作流图谱的潜力。

**高阶函数替代循环节点**。Map、Reduce、Filter 等函数可以优雅地替代传统画布上的迭代器节点。

## 快速开始

```bash
cd py
python3 workflow_server.py
```

打开浏览器访问 http://localhost:8080 ，即可使用内置的 Web IDE 编辑和运行脚本。

运行前需要在页面顶部输入你的 API Key（兼容 DeepSeek/OpenAI 格式），用于调用大模型。

## 语法指南

本解释器支持**两种书写风格**：标准 Lisp 括号 `()` 和中文方括号 `【】`，两者完全等价。

### 基础语法

```lisp
;; 标准写法
(print "Hello World")

;; 中文写法（推荐，更易读）
【打印 "Hello World"】
```

所有关键字都有对应的中文别名，可以混用。`【】` 会被解释器自动转换为 `()`。

### 注释

```lisp
;; 这是注释，以分号开头
```

## 语法详解

### 1. 定义变量

```lisp
【定义 名字 "张三"】
【定义 年龄 25】
【定义 列表 【序列 "苹果" "香蕉" "橙子"】】
```

### 2. 定义函数

```lisp
;; 语法：【定义 【函数名 参数1 参数2 ...】 函数体】
【定义 【问候 名字】
  【打印 "你好，" 名字 "！"】】

【问候 "张三"】  ;; 输出: 你好，张三！
```

### 3. 条件判断

```lisp
;; 语法：【如果 条件 真值 假值】
【如果 【> 分数 60】 "及格" "不及格"】

;; 嵌套条件
【如果 【> 分数 90】 "优秀"
  【如果 【> 分数 60】 "及格"
    "不及格"】】
```

### 4. 局部变量（令/let）

```lisp
;; 语法：【令 【【变量1 值1】【变量2 值2】】 表达式体】
【令 【【x 10】【y 20】】
  【打印 【+ x y】】】  ;; 输出: 30
```

### 5. 匿名函数（道/lambda）

```lisp
;; 语法：【道 【参数列表】 函数体】
【定义 加倍 【道 【n】 【* n 2】】】
【加倍 5】  ;; 结果: 10
```

### 6. 顺序执行（开始/begin）

```lisp
【开始
  【打印 "第一步"】
  【打印 "第二步"】
  【打印 "第三步"】】
```

### 7. 高阶函数

```lisp
;; 映射（map）：对列表每项应用函数
【映射 【道 【x】 【* x 2】】 【序列 1 2 3 4】】
;; 结果: [2, 4, 6, 8]

;; 过滤（filter）：保留满足条件的元素
【过滤 【道 【x】 【> x 3】】 【序列 1 2 3 4 5】】
;; 结果: [4, 5]

;; 归约（reduce）：将列表归约为单值
【归约 【道 【累计 当前】 【+ 累计 当前】】 0 【序列 1 2 3 4】】
;; 结果: 10
```

### 8. 管道操作（->）

将值依次通过多个函数，前一步的输出作为下一步的输入：

```lisp
【-> "  Hello World  "
    【文本裁剪】
    【打印】】
```

### 9. 模板格式化

使用 `{{name}}` 命名占位符，参数按 key-value 对传入：

```lisp
【定义 模板 "你好，{{名字}}！你今年{{年龄}}岁。"】
【格式化 模板 "名字" "张三" "年龄" "25"】
;; 结果: "你好，张三！你今年25岁。"
```

### 10. 调用大模型

```lisp
;; 直接调用
【定义 回复 【调用模型 "请用一句话解释什么是递归"】】
【打印 回复】

;; 配合模板使用
【定义 提示词 "请为《{{标题}}》写一段简介"】
【定义 结果 【调用模型 【格式化 提示词 "标题" "Lisp入门"】】】
```

### 11. 引用外部文件

将 Prompt 等长文本放在单独文件中，用 `读文件` 引用：

```lisp
;; 读取 prompts/大纲生成.txt 的内容作为提示词
【定义 提示词 【读文件 "prompts/大纲生成.txt"】】
【调用模型 【格式化 提示词 "标题" "AI简史"】】
```

## 完整工作流示例

### 文章生成（函数式风格）

```lisp
【定义 标题 "庄子的人生感悟"】
【定义 扩写提示词 【读文件 "prompts/章节扩写.txt"】】

;; 手动定义大纲
【定义 大纲 【序列 "得失的故事" "困境的故事" "选择的故事"】】

;; 逐章扩写：映射每个章节名 → 调用模型 → 格式化输出
【定义 文章列表
  【映射 【道 【章节名】
           【文本拼接 "## " 章节名 "\n\n"
             【去除思考 【调用模型 【格式化 扩写提示词 "标题" 标题 "章节" 章节名】】】】】
        大纲】】

;; 合并为完整文章
【打印 【文本拼接 "# " 标题 "\n\n" 【文本连接 "\n\n" 文章列表】】】
```

### Pipeline 模式（reduce 组合）

```lisp
;; 每个节点：接收上下文字典 → 返回更新后的字典
【定义 【注入标题 ctx】 【赋值 ctx "title" "Lisp入门"】】
【定义 【生成大纲 ctx】 【赋值 ctx "outline" 【调用模型 ...】】】
【定义 【解析大纲 ctx】 【赋值 ctx "sections" 【提取JSON ...】】】

;; 用归约串联所有节点，一行即工作流！
【定义 【执行工作流 ctx】
  【归约 【道 【当前 节点】 【节点 当前】】 ctx
    【序列 注入标题 生成大纲 解析大纲 逐章扩写 拼接文章】】】

【执行工作流 【字典】】
```

### 飞书通知（条件分支）

```lisp
【定义 【处理反馈 反馈】
  【令 【【情感 【调用模型 【格式化 情感提示词 "反馈" 反馈】】】】
    【如果 【文本开头 "positive" 情感】
      【发送飞书 "✅ 正向反馈" 反馈】
      【开始
        【令 【【类型 【调用模型 【格式化 分类提示词 "反馈" 反馈】】】】
          【发送飞书 【文本拼接 "❌ 负向-" 类型】 反馈】】】】】】
```

## 已知限制

1. **let 绑定互不可见**：同一个 `令` 的多个绑定中，后面的不能引用前面的变量。需要嵌套：
   ```lisp
   ;; ❌ 错误
   【令 【【x 1】【y 【+ x 1】】】 ...】

   ;; ✅ 正确：嵌套
   【令 【【x 1】】
     【令 【【y 【+ x 1】】】 ...】】
   ```

2. **字符串比较**：建议用 `文本开头` / `文本包含` 代替 `=` 做字符串比较。
